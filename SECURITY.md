## Security Policy

### Supported versions

Security updates are applied to the latest version on the default branch.

### Reporting a vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Instead:

- If GitHub Security Advisories are enabled for this repo, please use **“Report a vulnerability”** on the repository’s **Security** tab.
- Otherwise, open a GitHub issue with minimal details and ask maintainers for a private reporting channel.

### What to include

- A clear description of the issue and impact
- Steps to reproduce (proof-of-concept if available)
- Affected versions/commit SHA (if known)
- Any relevant logs or screenshots (redact secrets)
